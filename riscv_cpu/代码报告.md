# RISC-V CPU 代码报告

## 1. 项目总体说明

该工程是一个教学型/实验型的简化 RISC-V CPU，实现方式接近“分模块数据通路 + 组合控制”的结构。顶层模块为 `riscv`，通过 `ControlUnit` 产生控制信号，驱动 `PC`、`IM`、`IR`、`RF`、`ALU`、`DM`、`NPC`、`EXT` 以及多个多路选择器完成取指、译码、执行、访存、写回。

当前工程仍属于“未完全实现”状态，已支持部分 RV32I 指令路径（如 R-type 算术逻辑、`addi`/`ori`、`lw`、`sw`、`beq`/`bne`、`jal`/`jalr` 的基础控制）。

---

## 2. include 头文件报错原因分析

### 2.1 根因

原始工程把宏定义文件放在 `rtl/includes/` 下，但各 Verilog 文件普遍使用：

- `` `include "ctrl_signal_def.v" ``
- `` `include "instruction_def.v" ``
- `` `include "global_def.v" ``

如果仿真器/综合器没有正确添加 include 搜索路径（如 `+incdir+rtl/includes`），就会报 “include file not found”。

### 2.2 本次修复策略

采用“兼容式修复”，避免依赖 IDE 或仿真命令参数：

1. 在 `rtl/` 目录下新增同名头文件：
   - `rtl/ctrl_signal_def.v`
   - `rtl/instruction_def.v`
   - `rtl/global_def.v`
2. 在工程根目录再新增同名转发头文件：
   - `ctrl_signal_def.v`
   - `instruction_def.v`
   - `global_def.v`

这样即使不同工具以不同目录为当前工作目录，也能找到宏定义。

## 3. 已发现并修复的问题（代码级）

1. `ALU.v` 中 `ALU` 模块为空（功能缺失）
   - 已补充 `ADD/SUB/AND/OR/XOR/SLL/SRL/SRA` 运算。
   - 已补充 `zero` 标志输出逻辑。

2. `ControlUnit.v` 中模块声明缺少分号（语法错误）且控制逻辑缺失
   - 已修复声明语法。
   - 已新增组合控制逻辑和默认控制值。

3. `MUX_3to1_B.v` 缺失分号（语法错误）
   - 已修复 `ALUSrcB_else` 分支行末分号。

4. `MUX_2to1_A.v` 的 `Y` 端口位宽为 `5` 位，与顶层连接 `32'h0` 不匹配
   - 已改为 `32` 位，输出逻辑改为直接二选一。

5. `MUX_3to1_LMD.v` 的 `Z` 端口位宽为 `[31:2]`，与 `PCA4[31:0]` 不匹配
   - 已改为 `[31:0]`。

6. `riscv.v` 顶层缺少宏头文件 include
   - 已新增 `ctrl_signal_def.v` 与 `instruction_def.v` 的 include。

7. `riscv.v` 中 `NPC.rs` 连接成 `RD1[31:2]`（位宽不匹配）
   - 已修复为 `RD1`。

8. `instruction_def.v` 中 `INSTR_SW_OP` 定义错误（写成 I-type 的 `0010011`）
   - 已修正为 `0100011`。

9. `IM.v` 在 `InsMemRW==0` 时未定义输出行为
   - 已补充 `Ins <= 32'b0`，避免不确定输出传播。

10. `RF.v` 中维护 `x0` 的 `always @(clk)` 写法不规范
    - 已合并到 `always @(posedge clk)` 中，逻辑更稳定。

## 4. 各 Verilog 文件详细说明

## 4.1 顶层与核心控制

### 4.1.1 `rtl/riscv.v`
- 作用：CPU 顶层集成，连接所有子模块，完成数据通路组织。
- 关键输入：
  - `clk`：时钟
  - `rst`：复位
- 关键内部信号：
  - 控制信号：`RFWrite`、`DMCtrl`、`PCWrite`、`IRWrite`、`InsMemRW`、`ExtSel`、`ALUSrcA`、`ALUSrcB`、`NPCOp`、`WDSel`、`RegSel`、`ALUOp`
  - 指令字段：`opcode`、`Funct3`、`Funct7`、`rs1`、`rs2`、`rd`、`Imm12`
  - 跳转/分支偏移：`Offset`、`Offset20`
- 说明：负责从指令中拆字段，并将其送入控制单元与执行路径。

### 4.1.2 `rtl/ControlUnit.v`
- 作用：根据 `opcode/Funct3/Funct7/zero` 生成控制信号。
- 输入参数：
  - `rst`、`clk`（当前版本中主要使用组合逻辑，`clk/rst`用于接口兼容）
  - `zero`：来自 `ALU` 的零标志
  - `opcode`、`Funct7`、`Funct3`：译码字段
- 输出参数：
  - `PCWrite`：是否更新 PC
  - `InsMemRW`：是否允许取指
  - `IRWrite`：是否写指令寄存器
  - `RFWrite`：是否写寄存器堆
  - `DMCtrl`：数据存储器读写控制
  - `ExtSel`：立即数扩展方式
  - `ALUSrcA`、`ALUSrcB`：ALU 输入选择
  - `RegSel`：写回目标寄存器号选择
  - `NPCOp`：下一 PC 选择
  - `WDSel`：写回数据选择
  - `ALUOp`：ALU 运算类型
- 说明：已实现基础指令集的控制分派。

---

## 4.2 执行与运算相关模块

### 4.2.1 `rtl/ALU.v`
- 作用：执行算术/逻辑/移位运算。
- 输入：
  - `A[31:0]`、`B[31:0]`
  - `ALUOp[3:0]`
- 输出：
  - `ALU_result[31:0]`
  - `zero`（`ALU_result==0`）
- 当前支持：`ADD/SUB/AND/OR/XOR/SLL/SRL/SRA`。

### 4.2.2 `rtl/EXT.v`
- 作用：12 位立即数扩展到 32 位。
- 输入：
  - `imm_in[11:0]`
  - `ExtSel`
- 输出：
  - `imm_out[31:0]`
- 扩展模式：
  - `ExtSel_ZERO`：零扩展
  - `ExtSel_SIGNED`：符号扩展

### 4.2.3 `rtl/MUX_2to1_A.v`
- 作用：选择 ALU 输入 `A`。
- 输入：
  - `X[31:0]`
  - `Y[31:0]`
  - `control`
- 输出：
  - `out[31:0]`

### 4.2.4 `rtl/MUX_3to1_B.v`
- 作用：选择 ALU 输入 `B`（寄存器值/立即数/偏移量）。
- 输入：
  - `X[31:0]`
  - `Y[31:0]`
  - `Z[11:0]`
  - `control[1:0]`
- 输出：
  - `out[31:0]`（有符号）

### 4.2.5 `rtl/MUX_3to1_LMD.v`
- 作用：选择写回寄存器的数据来源（ALU/内存/PC+4）。
- 输入：
  - `X[31:0]`：ALU
  - `Y[31:0]`：DM
  - `Z[31:0]`：PCA4
  - `control[1:0]`
- 输出：
  - `out[31:0]`

### 4.2.6 `rtl/MUX_3to1.v`
- 作用：选择写寄存器地址（`rd/0/31`）。
- 输入：
  - `X[4:0]`、`Y[4:0]`、`Z[4:0]`
  - `control[1:0]`
- 输出：
  - `out[4:0]`

---

## 4.3 程序计数器与指令通路

### 4.3.1 `rtl/PC.v`
- 作用：程序计数器寄存器。
- 输入：
  - `clk`、`rst`
  - `PCWrite`
  - `NPC[31:0]`
- 输出：
  - `PC[31:0]`
- 行为：复位清零；`PCWrite=1` 时更新为 `NPC`。

### 4.3.2 `rtl/NPC.v`
- 作用：计算下一条指令地址。
- 输入：
  - `NPCOp[1:0]`
  - `Offset12[12:1]`（分支偏移）
  - `Offset20[20:1]`（JAL 偏移）
  - `PC[31:0]`
  - `rs[31:0]`
- 输出：
  - `PCA4[31:0]`
  - `NPC[31:0]`
- 行为：根据 `NPCOp` 在 `PC+4`、`PC+Offset`、`rs` 等路径中选择。

### 4.3.3 `rtl/IM.v`
- 作用：指令存储器（简化模型）。
- 输入：
  - `InsMemRW`
  - `addr[11:2]`
- 输出：
  - `Ins[31:0]`
- 说明：当前为组合读风格，`InsMemRW=0` 时输出 0。

### 4.3.4 `rtl/IR.v`
- 作用：指令寄存器。
- 输入：
  - `in_ins[31:0]`
  - `clk`
  - `IRWrite`
- 输出：
  - `out_ins[31:0]`
- 行为：时钟上升沿在 `IRWrite=1` 时锁存指令。

---

## 4.4 数据通路存储与寄存模块

### 4.4.1 `rtl/RF.v`
- 作用：32x32 寄存器堆。
- 输入：
  - 读地址：`RR1[4:0]`、`RR2[4:0]`
  - 写地址：`WR[4:0]`
  - 写数据：`WD[31:0]`
  - `RFWrite`、`clk`
- 输出：
  - `RD1[31:0]`、`RD2[31:0]`
- 行为：异步读，同步写；每个时钟上升沿强制 `x0=0`。

### 4.4.2 `rtl/DM.v`
- 作用：数据存储器（简化模型）。
- 输入：
  - `Addr[11:2]`
  - `WD[31:0]`
  - `clk`
  - `DMCtrl`
- 输出：
  - `RD[31:0]`
- 行为：时钟上升沿写或读（同步读写模型）。

### 4.4.3 `rtl/Flopr.v`
- 作用：32 位通用寄存器（带异步复位）。
- 输入：
  - `clk`、`rst`
  - `in_data[31:0]`
- 输出：
  - `out_data[31:0]`
- 用途：锁存流水级/阶段间数据（如 `A`、`B`、`ALU_result`）。

## 4.5 头文件（宏定义）

### 4.5.1 `rtl/includes/ctrl_signal_def.v`
- 作用：定义控制信号编码（`NPCOp`、`ALUSrcA/B`、`ExtSel`、`ALUOp`、`RegSel`、`WDSel`、`DMCtrl`）。

### 4.5.2 `rtl/includes/instruction_def.v`
- 作用：定义指令 `opcode/funct` 宏。
- 说明：`INSTR_SW_OP` 已修复为 `7'b0100011`。

### 4.5.3 `rtl/includes/global_def.v`
- 作用：全局宏（例如 `DEBUG`）。

### 4.5.4 兼容头文件（新增）
- `rtl/ctrl_signal_def.v`
- `rtl/instruction_def.v`
- `rtl/global_def.v`
- `ctrl_signal_def.v`
- `instruction_def.v`
- `global_def.v`

作用：兼容不同工具的 include 查找路径，降低“找不到头文件”的概率。

## 5. 目前仍建议后续完善的点

1. `NPC` 对 `jalr` 的规范行为应为 `(rs1 + imm) & ~1`，当前路径仍较简化。
2. `DM`/`IM` 可进一步明确初始化、字节使能、异步/同步读策略。
3. `ControlUnit` 可升级为更完整状态机（若目标是多周期 CPU）。
4. 建议补充 testbench 与基础指令回归测试（算术、访存、分支、跳转）。
